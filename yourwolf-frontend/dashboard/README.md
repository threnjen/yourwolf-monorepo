There are two big problems with the dashboard right now:
- Handling iterables is hard.  When we come across nested iterables, we're not able to create or layout the interaction buttons in a way that (1) works and (2) is intuitive.  Ideally, we'd have some way to separate and display dictionary entries (OR conditions) on the dashboard, and a way to sseparate and display list items.
- I have NO idea how to load abilities.  Right now abilities are stored as subclasses, but I'm pretty sure that's incorrect.  Right now this causes the dashboard to try to build an ability from scratch rather than pick from a pre-selected list.  I have no idea how to even begin to change this, and it seems like a very hard problem to solve while trying to keep things general.  We could have a single "if ability do something different" option, but that does two things:  forces business logic into the dashboard, and locks us into abilities existing as they do today.  Maybe that's a good holdover solution?
